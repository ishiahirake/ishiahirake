Material Design 颜色系统主要包含有如下颜色:

- 主色(Primary color)
- 辅色(Secondary color)
- 浅色或深色的变体

基本原则:

- 层次(Hierarchical): 颜色表明哪些元素是可交互的, 相对其他元素的关系已经重要程度.
- 清晰易读(Legible): 文字和重要的元素, 出现再有颜色的背景之上时应当满足清晰易读的原则.
- 富有表现(Expressive): 在令人难忘的时刻(memorable moments)展示品牌色彩，以增强品牌的风格。

## 调色板

### 主色

**主色** 是最经常在显示的颜色. 可以有深色或者浅色的变体.

### 辅色

**辅色** 提供了更多强调或者区分产品的途径. 辅色是可选的, 应尽可能少的在选定的 UI 部分使用.
跟主色类似, 辅色也可以浅色或者深色的变体.

辅色适合于:

- Floating action buttons
- Selection controls, like sliders and switches
- 高亮选择的文字
- 进度条
- 链接和标题

### 表面, 背景和错误色

与主色、辅色不同, 表面、背景以及错误色并不用于表示品牌:

- 表面色(Surface color): 影响组件的表面, 如 卡片(Card), sheets 和菜单
- 背景色(Background color): 显示在可滚动的元素的背后.
- 错误色(Error Color): 表示组件的错误, 如一个文本框的错误消息.

### 文字和图标颜色

#### 在...之上的颜色(On 颜色)

当文字或图标, 出现在主色或者辅色的表面之上时, 这些元素需要使用与表面易于区分辨识的颜色.
这种颜色称之为 **在...之上** 的颜色. 有如下的颜色:

- 主色之上(On Primary)
- 辅色之上(On Secondary)
- 背景色之上(On Background)
- 表面色之上(On Surface)
- 错误色之上(On Error)

### 可选色

除了主色跟辅色之外, 也可以包含其他的可选色. 可选色(Alternative colors)可以用于区分 UI 的不同部分.
可选色适用于:

- 有浅色和深色主题的应用
- 不同的部分有不同主题的应用
- 属于产品套件中的一部分的应用
